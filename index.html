How This HTML Works

This page is built as a simple interactive gallery. The top part of the page shows a featured project, and the row of thumbnails underneath lets the user choose a different project to display.

How the page is organized

  1. The <head> contains setup information like the page title, CSS links, and JavaScript links.
  2. The <body> contains the visible content of the page.
  3. The featured project section contains the large image, title, and description.
  4. The thumbnail gallery contains smaller clickable images.

How the interactive part works

The large image at the top has the ID featured-image. The title has the ID project-title, and the description has the ID project-description. JavaScript uses these IDs to find and update those parts of the page.

Each thumbnail stores extra information in its attributes:

  • src = the image file
  • data-title = the project title
  • data-description = the project description

When a thumbnail is clicked, the JavaScript reads those values and uses them to:

  • change the featured image
  • change the project title
  • change the project description
  • update the active thumbnail styling

Why use IDs in the featured area?

An ID should be unique on the page. Since there is only one featured image, one featured title, and one featured description, IDs are a good choice.

Why use classes on the thumbnails?

Classes can be reused on multiple elements. Since there are several thumbnail images, using the class thumb makes sense.

Why use custom data attributes?

The attributes data-title and data-description store extra information directly inside the HTML. This makes it easy for JavaScript to read that information when a thumbnail is clicked.

Glossary of HTML Terms

<!DOCTYPE html>
Tells the browser this is an HTML5 document.
<head>
Contains setup information that is not displayed directly on the page.
<body>
Contains the visible content of the page.
<title>
Sets the text shown in the browser tab.
<link>
Connects an external CSS file to the page.
<script>
Connects a JavaScript file to the page.
<main>
Represents the main content area of the page.
<header>
Represents introductory content such as headings and instructions.
<section>
Groups related content together.
<div>
A general container used to group elements together.
<img>
Displays an image.
class
Used for styling or scripting multiple elements.
id
Used to uniquely identify one specific element.
src
The image file path used by an image element.
alt
Alternative text describing an image.
data-title
A custom data attribute storing the project title.
data-description
A custom data attribute storing the project description.

Important IDs and Classes

  • wrapper - Main layout container for the page
  • featured-project - Styles the featured project section
  • featured-image - The large image that changes when a thumbnail is clicked
  • featured-text - Wraps the featured title and description
  • project-title - The title that JavaScript updates
  • project-description - The description that JavaScript updates
  • thumbnail-gallery - The section containing the thumbnails
  • thumb-grid - The layout container for the thumbnails
  • thumb - Applied to every thumbnail image
  • active - Marks the thumbnail that is currently selected
